# 前端与可视化

## 技术选型

MVP 前端使用：

- React
- Vite
- pnpm
- D3.js

## 角色边界

前端仅作为展示层。

Python 脚本产出权威数据工件：

- 原始新闻（raw news）
- 清洗后新闻（cleaned news）
- 结构化新闻（structured news）
- 聚类事件（clustered events）
- 重要性评分（importance scores）
- 可视化数据（visualization data）
- 日报（daily report）
- 质量检查结果（quality check result）

React 应用读取这些工件并渲染仪表盘。它不得调用 LLM、创建事实、修改排名或生成不可追溯的结论。

## 仪表盘数据契约

管线将前端可用数据导出到：

```text
frontend/public/data/latest.json
```

仪表盘可包含：

- 来源类型分布
- 事件类型分布
- 行业领域分布
- 顶级事件排名
- 风险与机会矩阵
- 报告摘要及证据链接

## 证据要求

顶级事件、趋势、风险和机会必须通过以下方式保持可追溯性：

- `news_id`
- `event_id`
- 来源名称
- 来源 URL
- 关键事实
- 证据摘要

## 可视化图表

| 图表 | 类型 | 说明 |
|:---|:---|:---|
| 来源类型分布 | 环形图（Donut） | 展示证据来自研究、官方、媒体还是社区来源 |
| 事件类型分布 | 环形图（Donut） | 展示用于聚类和排名的事件类别 |
| 风险/机会矩阵 | 散点矩阵（Scatter） | 每个点代表一个事件，颜色=事件类型，大小=置信度。支持缩放、拖拽、右键复位 |

### 矩阵图交互

- **滚轮缩放**：放大查看密集区域细节
- **拖拽平移**：移动视图到感兴趣的区域
- **右键/双击复位**：恢复初始视图
- **悬浮提示**：浮动 tooltip 显示事件名、风险/机会分数
- **力导向防重叠**：相近的数据点自动推开

## 仓库规则

- 提交前端源码文件。
- 安装依赖后提交 `pnpm-lock.yaml`。
- 不要提交 `node_modules/`。
- 除非后续交付要求明确要求静态构建输出，否则不要将构建后的前端资源作为源码提交。
