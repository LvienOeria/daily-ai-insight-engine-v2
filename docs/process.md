# Core Process

## Workflow Overview

```text
1. Load sources from config/sources.json
2. Fetch each source (API / RSS / HTTP scrape)
3. Evaluate source quality → source_evaluation.json
4. Clean & validate → cleaned_news.json (dedup, date filter, quality score)
5. Structure with DeepSeek → structured_news.json (entities, event_type, risk_tags, etc.)
6. Cluster into events with DeepSeek → clustered_events.json
7. Score event importance → importance_scores.json
8. Generate visualization data → visualization_data.json
9. Generate daily report with DeepSeek → daily_ai_report.md
10. Run quality check → quality_check.json
11. Export frontend payload → frontend/public/data/latest.json
```

## Step Details

### 1-2. Fetch

- **arXiv**: API call with query `cat:cs.AI OR cat:cs.CL OR cat:cs.LG`, max 20 results
- **RSS**: `feedparser` parses XML, extracts title/summary/date/URL
- **HTTP scrape**: fetch homepage → extract article links → fetch each article → extract title/summary/date from HTML

RSS feeds are cached for 1 hour in `data/cache/`.

### 3-4. Clean

- Normalize dates to `Asia/Shanghai`
- Remove items outside the 3-day reporting window
- Deduplicate by URL
- Filter items missing required fields (title, source, published_at)
- Cap at `max_items` (default 20)

### 5. Structure

DeepSeek (or mock keyword matching) extracts from each cleaned item:

- `entities`, `technologies`, `event_type`, `industry_area`
- `key_facts`, `sentiment`, `impact_scope`
- `risk_tags`, `opportunity_tags`, `importance_score`, `confidence`, `evidence`

Batch size: 5 items per API call.

### 6-7. Cluster & Score

DeepSeek groups related news into events. Then a deterministic scoring model ranks events by:
impact scope, source authority, novelty, multi-source support, technical/business impact, risk/opportunity level, recency.

### 8-9. Visualize & Report

Visualization data is computed from structured items and events. The daily report is generated by DeepSeek in professional Chinese with 7 required sections.

### 10-11. Quality Check & Export

Quality check validates item counts, field completeness, evidence traceability, report sections, and documentation presence. All artifacts are exported to `frontend/public/data/latest.json` for the React dashboard.

## Operating Principle

Rules are defined by humans, deterministic processing ensures stability, LLM handles semantic understanding, and quality checks flag issues before the report is finalized.
