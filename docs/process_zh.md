# 核心流程

## 目的

本文档描述从原始信息到最终每日 AI 洞察报告的端到端工作流。

## 工作流概览

```text
1. 定义报告范围
2. 构建候选源池
3. 评估候选源
4. 选择 MVP 数据源
5. 采集 10-20 条原始 AI 新闻
6. 规范化原始数据
7. 清洗并验证数据
8. 使用 LLM 抽取结构化字段
9. 验证结构化输出
10. 将新闻聚类为事件
11. 评分事件重要性
12. 生成可视化数据
13. 生成日报
14. 运行质量检查
15. 完成文档
```

## 步骤 1：定义报告范围

确定 MVP 的焦点：

- 报告时间窗口
- AI 主题覆盖范围
- 中英文源比例
- 目标受众
- 输出格式
- 手动与 API/RSS 数据采集的边界

## 步骤 2：构建候选源池

列出跨以下类型的可能来源：

- 新闻聚合平台
- 官方 AI 公司源
- 研究源
- 技术社区
- 中文 AI 媒体
- 手动静态数据

## 步骤 3：评估候选源

使用 `data-source-evaluation` skill。

记录：

- 字段完整度
- AI 主题相关性
- 可信度
- 稳定性
- 成本或访问限制
- 推荐层级

## 步骤 4：选择 MVP 源

选择能够为 MVP 产出足够高质量数据的源。

优先选择小而均衡的源集合，而非庞大但不稳定的源列表。

## 步骤 5：采集原始数据

采集 10-20 条近期 AI 相关条目。

每条主要条目应包含：

- 标题
- 摘要/正文/描述
- 来源
- 发布时间
- URL 或可追溯的唯一标识符（尽可能提供）

## 步骤 6：规范化原始数据

将不同的 API/RSS/手动字段映射到统一的 RawNewsItem Schema。

尽可能保留原始载荷或原文。

## 步骤 7：清洗并验证数据

应用确定性检查：

- 必填字段检查
- 日期规范化
- 来源规范化
- 语言标记
- 去重
- AI 相关性过滤
- 质量评分

## 步骤 8：结构化抽取

使用 `news-structuring` skill。

抽取：

- 实体（entities）
- 技术（technologies）
- 事件类型（event_type）
- 行业领域（industry_area）
- 关键事实（key_facts）
- 情感（sentiment）
- 影响范围（impact_scope）
- 风险标签（risk_tags）
- 机会标签（opportunity_tags）
- 重要性评分（importance_score）
- 置信度（confidence）
- 证据（evidence）

## 步骤 9：验证结构化输出

检查：

- JSON 有效性
- 必填字段
- 枚举有效性
- 分数范围
- 证据存在性
- 低置信度条目

## 步骤 10：事件聚类

使用 `event-clustering` skill。

通过以下方式将相关新闻归类为事件：

- URL/标题重复检查
- 实体重叠
- 事件类型
- 关键事实
- 语义相似性

## 步骤 11：重要性评分

使用 `importance-scoring` skill。

按以下维度对事件进行排名：

- 影响范围
- 来源权威性
- 新颖性
- 多源支持度
- 技术影响
- 商业影响
- 风险水平
- 机会水平
- 时效性

选择 Top 3-5 事件。

## 步骤 12：可视化

从结构化数据生成图表。

推荐图表：

- 来源类型分布
- 事件类型分布
- 技术主题分布
- 顶级事件重要性排名
- 风险/机会矩阵

在本项目中，可视化通过 React + Vite 前端结合 D3.js 实现。Python 管线输出为权威数据源，并将仪表盘数据导出到 `frontend/public/data/latest.json`。

前端规则：

- 前端仅展示已生成的工件。
- 不得调用 LLM 或编造额外分析。
- 图表数据必须与 `data/visualization_data.json` 及排名事件工件一致。
- 顶级事件和主要结论的证据链接或 ID 必须保持可见。

## 步骤 13：日报生成

使用 `daily-report-generation` skill。

报告应包含：

- 今日概览
- Top 3-5 事件
- 深度分析
- 趋势判断
- 风险与机会提示
- 可视化说明
- 数据与方法说明

## 步骤 14：质量检查

使用 `report-quality-check` skill。

检查：

- 任务完整度
- 证据可追溯性
- 无支撑的论断
- Schema 有效性
- 可视化一致性
- 文档缺失

## 步骤 15：完成文档

更新：

- 数据源决策
- Schema 设计
- AI 使用方式
- 核心流程
- 已知局限

## 最终交付物

预期的 MVP 交付物：

- 源评估说明
- 原始或清洗后的数据文件
- 结构化数据文件
- 事件/排名数据文件
- 可视化输出
- 最终日报
- 文档文件

## 运作原则

人定义规则，确定性处理保障稳定性，LLM 处理语义理解，人工审查确认关键结论。
