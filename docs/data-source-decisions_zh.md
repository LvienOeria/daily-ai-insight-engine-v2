# 数据源决策

## 目的

本文档记录每日 AI 洞察引擎 MVP 的数据源选择过程。

目标是使源选择可解释、可复现。

## 源选择策略

MVP 数据采集优先使用 API、RSS 和手动整理的静态数据。

对于中文语言兼容性，MVP 还可使用 DeepSeek 网络搜索作为受限的采集路径。该路径仅限于以下站点的网页：

- 量子位（`qbitai.com`）
- 机器之心（`jiqizhixin.com`）
- 知乎（`zhihu.com`）

DeepSeek 搜索结果不会自动被接受为核心数据。工作流必须首先尝试检索，检查实际字段完整度，然后按照与 API 或 RSS 源相同的必填字段和可信度规则评估每个源/结果。

不优先使用爬虫采集的原因：

- 任务允许手动整理的静态数据。
- 主要评估重点是结构化处理、分析逻辑和验证。
- 新闻网站的 HTML 结构差异很大。
- 爬虫增加了维护、访问、反爬和合规复杂性。
- API 和 RSS 源在获取标题、摘要、来源、发布时间和 URL 方面更加稳定。

## 必填字段

主分析条目应包含：

- 标题
- 摘要/正文/描述
- 来源
- 发布时间
- URL 或可追溯的唯一标识符（尽可能提供）

缺少标题、来源或发布时间的条目不应进入主分析集，除非使用可追溯的证据手动修复。

## 源分级

核心源（core）：

- 足够稳定且字段完整，可用于 MVP 分析。
- 用于主数据集。

辅助源（auxiliary）：

- 提供有用信号，但可能存在字段缺失或可靠性有限。
- 用于上下文或可选分析。

未来源（future）：

- 有价值，但由于爬虫复杂性、登录、配额、成本或字段不稳定等原因不适合 MVP。

拒绝源（reject）：

- 缺少关键字段、AI 相关性低、无法验证或过于不稳定。

## 中文网络搜索规则

使用 DeepSeek 网络搜索中文源时：

- 将查询限制在白名单站点：量子位、机器之心和知乎。
- 保存查询文本、收集时间、返回的标题、URL、来源/站点、摘要/正文以及任何可见的发布日期。
- 除非结果包含标题、来源/站点、发布日期（或可追溯的可见日期）和摘要/正文，否则不将其纳入主分析集。
- 如果发布日期缺失或模糊，标记为缺失，并将该条目排除在顶级事件的主要证据之外，除非使用可追溯的证据手动修复。
- 将知乎主要视为社区或舆情信号。除非具体条目本身完整且可追溯，否则不应将其作为重大事实事件的唯一来源。
- 在字段验证和源评估通过之前，将搜索输出与清洗数据分开保存。

## 已知局限

- MVP 数据集较小，通常为 10-20 条。
- API 和 RSS 源可能不提供完整的正文文本。
- 手动整理的项目提高了质量但降低了自动化程度。
- 数据集可能无法代表所有平台上的完整舆论。
- 中文网络搜索的输出质量取决于搜索结果中的字段和页面可访问性；观察结果必须在源选择之前进行评估。
