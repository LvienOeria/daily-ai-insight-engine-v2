# 数据源决策

## 目的

本文档记录每日 AI 洞察引擎 MVP 的数据源选择过程。

目标是使源选择可解释、可复现。

## 源选择策略

MVP 数据采集使用 API 和 RSS 源。

对于中文语言兼容性，评估了三个候选源：

- **量子位**（`qbitai.com`）— ✅ 通过直连 HTTP 抓取可用
- **机器之心**（`jiqizhixin.com`）— ❌ 从非中国 IP 发起 SSL 握手超时
- **知乎**（`zhihu.com`）— ❌ 跳转至登录页面，无法抓取

同时测试了 DeepSeek API 的 `enable_search` 参数作为替代路径（利用 DeepSeek 位于中国的服务器作为代理），但 API 实际不会触发网络搜索——该功能仅在 DeepSeek 网页聊天界面可用，Chat Completions API 层未开放。

**结论**：目前仅有量子位可为 MVP 提供可用的中文数据。机器之心和知乎已禁用，标注为"未来"源，待有中国侧代理或 DeepSeek API 开放网络搜索后启用。

不优先使用爬虫采集的原因：

- 主要评估重点是结构化处理、分析逻辑和验证。
- 新闻网站的 HTML 结构差异很大。
- 爬虫增加了维护、访问、反爬和合规复杂性。
- API 和 RSS 源在获取标题、摘要、来源、发布时间和 URL 方面更加稳定。

## 必填字段

主分析条目应包含：

- 标题
- 摘要/正文/描述
- 来源
- 发布时间
- URL 或可追溯的唯一标识符（尽可能提供）

缺少标题、来源或发布时间的条目不应进入主分析集。

## 中文源评估结果

三个中文源经实测后的评估结论：

| 源 | 类型 | 访问方式 | 结果 | 层级 | 原因 |
|---|---|---|---|---|---|
| 量子位 | 媒体 | 直连 HTTP 抓取（回退） | ✅ 获取 10 条，字段完整，日期可从 URL 提取 | core | 非中国 IP 可访问；文章页返回 200；标题/摘要/日期可提取 |
| 机器之心 | 媒体 | 直连 HTTP / DeepSeek API | ❌ SSL 握手超时；DS API 网络搜索不可用 | future | 从非中国 IP 被 geo-block；DeepSeek API 的 enable_search 不触发实际请求 |
| 知乎 | 社区 | 直连 HTTP / DeepSeek API | ❌ 跳转至登录页；DS API 网络搜索不可用 | future | 需认证；不适合自动化采集 |

## 源分级

核心源（core）：

- 足够稳定且字段完整，可用于 MVP 分析。当前：arXiv、OpenAI News RSS、量子位。
- 用于主数据集。

辅助源（auxiliary）：

- 提供有用信号，但可能存在字段缺失或可靠性有限。当前：Hacker News。
- 用于上下文或可选分析。

未来源（future）：

- 有价值但因访问限制暂不可用。当前：机器之心、知乎、Google DeepMind RSS。

拒绝源（reject）：

- 缺少关键字段、AI 相关性低、无法验证或过于不稳定。

## 中文网络搜索规则

使用 DeepSeek 网络搜索中文源时：

- 将查询限制在白名单站点：量子位、机器之心和知乎。
- 保存查询文本、收集时间、返回的标题、URL、来源/站点、摘要/正文以及任何可见的发布日期。
- 除非结果包含标题、来源/站点、发布日期（或可追溯的可见日期）和摘要/正文，否则不将其纳入主分析集。
- 如果发布日期缺失或模糊，标记为缺失，并将该条目排除在顶级事件的主要证据之外。
- 将知乎主要视为社区或舆情信号。除非具体条目本身完整且可追溯，否则不应将其作为重大事实事件的唯一来源。
- 在字段验证和源评估通过之前，将搜索输出与清洗数据分开保存。

## 已知局限

- MVP 数据集较小，通常为 10-20 条。
- API 和 RSS 源可能不提供完整的正文文本。
- 数据集可能无法代表所有平台上的完整舆论。
- 中文网络搜索的输出质量取决于搜索结果中的字段和页面可访问性；观察结果必须在源选择之前进行评估。
